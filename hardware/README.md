# hardware/

The physical machine: one passive backplane plus six plug-in cards, each in its
own directory holding the Eagle `.sch`/`.brd`, a schematic PDF, a chip-by-chip
README, and any board-specific design docs.

| Board | Directory | Function |
|-------|-----------|----------|
| Control / Microcode | [control-card/](control-card/README.md) | Clock, reset, sequencer, microcode EPROMs, IR, condition mux, front panel |
| Register Bank | [regbank-card/](regbank-card/README.md) | P0–P3 16-bit pointer registers, address-bus drivers |
| ALU | [alu-card/](alu-card/README.md) | A/B/T/T2 registers, 74181 ALU, shifter, FLAGS |
| Memory | [memory-card/](memory-card/README.md) | 28C256 EEPROM + 62256 SRAM, address decode |
| I/O | [io-card/](io-card/README.md) | Switches, LEDs, 6850 ACIA (RS-232) |
| CF-IDE | [cf-card/](cf-card/README.md) | CompactFlash in 8-bit True IDE mode at `$FF10–$FF17` |
| Backplane | [backplane/](backplane/p8x-backplane-design.md) | 10-slot passive 96-pin DIN 41612 bus (rev C2) |

`p8x-bom.csv` — the orderable bill of materials for the whole machine
(ICs, sockets, connectors, passives), generated by
[`../generators/gen_bom.py`](../generators/).

## Regenerating boards

**Generators are canon** — never hand-edit `.sch`/`.brd` files. They are written
by [`../generators/gen_eagle.py`](../generators/), which emits per-board
subdirectories *relative to the current directory*, so run the generators from
**this** directory:

```sh
python3 ../generators/gen_eagle.py                # all .sch/.brd (hardware/<board>/)
python3 ../generators/render_traditional_auto.py  # all 7 card schematic PDFs
python3 ../generators/render_board_pdf.py         # placement-view PDFs
```

See [../generators/README.md](../generators/README.md) for the full list and
[../docs/p8x-card-standards.md](../docs/p8x-card-standards.md) for the rules
every card follows.
