# LED Output Card (test)

> **Theory of operation:** [p8x-led-card-theory.md](p8x-led-card-theory.md) — deep walkthrough of inputs/outputs, signal flow, and logic.

A minimal memory-mapped **output** card: write a byte to `$FF0C` and its 8 bits
appear on 8 LEDs. Write-only (no readback). It was built primarily as a
**CAD-workflow trial** — a small, self-contained card to exercise the
generate → validate → render → fab pipeline — but it is a complete, standards-
conforming design.

## At a glance

| | |
|---|---|
| Address | `$FF0C` (I/O page, write-only) |
| Function | latch the data byte; drive 8 LEDs from it |
| ICs | 6 (74HCT logic) |

## Chips

| Ref | Device | Role |
|-----|--------|------|
| U1 | 7430 | 8-input NAND — I/O page detect (`A8–A15` all high → `IOPG`) |
| U2 | 74138 | address decode: `A1–A3` select, enabled by `IOPG` + `A4=0`; `Y6` = `$FF0C` |
| U3 | 74138 | `DLD` field decode → `-MEMW` (write strobe) |
| U4 | 74HCT32 | OR gates: `WRSEL` (our-port write) and the latch clock `LCK` |
| U5 | 74HCT14 | inverter — `CLKBN` (the rest of the hex is spare, tied off) |
| U6 | 74374 | the 8-bit LED latch (`!OC` tied low — outputs always enabled) |

Plus the standard support parts: DIN96C edge connector `J1`, per-IC 100 nF
decoupling caps, an 8×330 Ω network `RL1` + 8-LED array `LA1`, a power LED, and a
write-activity LED.

## Use

```
POKE $FF0C, n      ; (monitor: deposit n at FF0C) → n's bit pattern on the LEDs
```

Each bit drives one LED (bit 0 = LED 0 … bit 7 = LED 7); a `1` lights the LED.

## Build

Generated by the canonical generator (run from `hardware/`):

```sh
python3 ../generators/gen_eagle.py                # writes led-card/*.sch + *.brd
python3 ../generators/render_traditional_auto.py  # writes the schematic PDF
```

**Generators are canon** — edit `generators/gen_eagle.py` (the `LED OUTPUT CARD`
section), never the `.sch`/`.brd` directly.

> **Note:** this is a scratch/test card and may be removed. Like the other I/O
> cards it decodes `A1–A4` but not `A5–A7`, so the port aliases every 32 bytes in
> the `$FFxx` page — acceptable for a test, worth tightening if it graduates to a
> permanent card.
